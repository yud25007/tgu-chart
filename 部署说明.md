# 服务器部署说明

本文件是公开仓库可提交版本。真实服务器别名、域名、Cloudflare 账号、VPC Service ID、Tunnel ID 和 token 请放在本地未提交配置中。

## 环境变量

复制 `.env.example` 为 `.env`，并填写真实值：

```bash
cp .env.example .env
```

至少需要设置：

```text
AUTH_PASSWORD=your-login-password
```

如果使用 `docker-compose.yml` 中的 Cloudflare Tunnel 容器，还需要设置：

```text
TGU_CHART_TUNNEL_TOKEN=your-cloudflare-tunnel-token
```

`.env` 不要提交到 Git。

## Docker 部署

在服务器项目目录运行：

```bash
docker compose up -d --build
```

健康检查：

```bash
curl http://127.0.0.1:18095/health
```

Compose 默认将数据持久化到：

```text
./data:/app/data
./output/doc:/app/output/doc
```

容器重建后，历史记录和已生成 Word 不会丢失。

## 同步代码

从本地同步到服务器时，不要覆盖服务器 `.env`、数据库和生成文件：

```bash
rsync -az --delete \
  --exclude '.env' \
  --exclude '.env.*' \
  --exclude '.DS_Store' \
  --exclude '__pycache__/' \
  --exclude '.wrangler/' \
  --exclude '.playwright-cli/' \
  --exclude 'data/' \
  --exclude 'output/' \
  ./ your-ssh-host:/path/to/project/
```

## Cloudflare Worker

真实 `wrangler.toml` 不提交。初始化时复制示例文件：

```bash
cp wrangler.example.toml wrangler.toml
```

填写真实的 `account_id`、自定义域名、zone 和 VPC Service ID 后部署：

```bash
npx wrangler deploy
```

## 公开仓库注意事项

- 不提交 `.env`。
- 不提交 `wrangler.toml`。
- 不提交 `.wrangler/`。
- 不提交 `data/`、`output/`。
- 不在文档中写真实学生姓名、学号、服务器地址、域名或 Cloudflare 资源 ID。
