# 开发日记

## 2026-06-27

### 批量删除历史记录

- 在历史记录区域新增多选能力，每条记录前增加复选框。
- 历史记录顶部新增“全选 / 取消全选”“批量删除”和已选数量显示。
- 后端新增 `/records/batch-delete` 接口，接收 `recordIds` 数组。
- 批量删除会同时删除 SQLite 中的记录和对应的 Word 输出文件。
- 单次批量删除最多允许 100 条，避免误传过大的请求。
- 删除结果会返回已删除 ID、已不存在 ID 和删除数量，前端据此刷新列表并提示结果。

### 安全与数据边界

- 批量删除接口沿用现有登录校验，未登录请求会被拒绝。
- 只允许正整数记录编号，空列表、非法编号和超限请求会返回错误。
- 保持 `.env`、真实 `wrangler.toml`、数据库、生成文件和临时截图不提交到 Git。
