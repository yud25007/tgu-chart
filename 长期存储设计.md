# 长期存储设计

## 当前阶段：本地单机存储

当前工具使用本地 SQLite 数据库：

```text
data/auto_form.sqlite3
```

每次点击“生成 Word”后，会保存：

- 活动名称
- 活动日期
- 加分类型
- 默认加分数量
- 名单明细：班级、姓名、学号、加分数量
- 生成的 Word 文件名
- 创建时间、更新时间

生成的 Word 文件仍保存到：

```text
output/doc/
```

网页端“历史记录”会从 SQLite 读取记录，支持载入、下载和删除。

## 数据表

### records

保存一次活动生成记录。

```text
id
activity_name
activity_date
credit_type
default_points
entry_count
output_filename
created_at
updated_at
```

### record_entries

保存活动下的名单明细。

```text
id
record_id
sort_order
class_name
name
student_id
points
```

### login_events

保存后台登录审计记录。

```text
id
ip
user_agent
success
session_id
created_at
```

### auth_sessions

保存登录会话与最近活跃时间，用于计算当前在线人数。

```text
session_id
ip
user_agent
created_at
last_seen_at
expires_at
logged_out_at
```

## 后续演进

### 阶段 2：草稿与复用

- 增加“保存草稿”，未生成 Word 也能长期保存。
- 增加“活动模板”，复用常见加分类型、默认学分、学院意见字段。
- 增加搜索和筛选：按日期、活动名称、班级、学号查找。

### 阶段 3：备份与导出

- 一键导出全部历史记录为 `.xlsx`。
- 一键打包 `data/` 和 `output/doc/`，方便换电脑。
- 启动时检测数据库和文件缺失，给出修复提示。

### 阶段 4：多人协作

- 后端换成局域网服务或云端数据库。
- 增加账号、角色、操作日志。
- Word 模板、生成记录、名单导入文件统一归档。

## 备份建议

定期备份这两个目录：

```text
data/
output/doc/
```

只要这两个目录还在，历史记录和已生成 Word 都能恢复。
